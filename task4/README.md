# Детектирование объектов с Transfer Learning

Проект по детектированию жестов языка жестов с использованием Transfer Learning на основе YOLOv8.

## Описание

Реализация задачи детектирования объектов (жестов языка жестов, буквы A-Z) с использованием предобученной модели YOLOv8.

**Transfer Learning:** использована предобученная модель YOLOv8 , которая дообучена на датасете жестов языка жестов.

## Структура проекта

```
├── convert_to_yolo.py       # Конвертация CSV → YOLO формат
├── train_yolo.py            # Обучение модели с transfer learning
├── test_model.py            # Тестирование модели
├── requirements.txt         # Зависимости
├── results/                 # Результаты
│   ├── result_*.jpg         # Визуализации детекций
│   └── detections.csv       # Результаты детекций
└── data_sample/             # Примеры данных
    ├── annotations_sample.csv
    └── image_*.jpg
```

## Установка

```bash
pip install -r requirements.txt
```

## Использование

### 1. Подготовка данных
```bash
python convert_to_yolo.py
```
Конвертирует аннотации из CSV формата в формат YOLO.

### 2. Обучение модели
```bash
python train_yolo.py
```
Обучает модель YOLOv8 с transfer learning на подготовленных данных.

### 3. Тестирование
```bash
python test_model.py
```
Тестирует обученную модель на тестовом наборе данных.

## Данные

Датасет: 1000 изображений жестов языка жестов с аннотациями.

Формат аннотаций (CSV):
```csv
filename,width,height,class,xmin,ymin,xmax,ymax
image_000000.jpg,512,512,A,88,0,283,165
```

Классы: буквы A-Z (26 классов).

## Результаты

Модель обучена на GPU (Google Colab) в течение 100 эпох.

### Метрики

- **mAP50**: 0.1077
- **mAP50-95**: 0.1035
- **Precision**: 0.0857
- **Recall**: 0.5896

### Анализ результатов

Низкие метрики объясняются следующими факторами:
1. большое количество классов 26 визуально похожих жестов
2. Малое количество данных на класс ~40 примеров на каждую букву
3. Сложность различения: жесты языка жестов очень похожи визуально

Модель технически работает корректно:
- Transfer learning применён правильно
- Loss стабильно снижается во время обучения
- Recall высокий (модель находит объекты)
- Precision низкий (много ложных детекций из-за путаницы между похожими классами)
